# Evidence Custody System (ECS)

| Field | Value |
|-------|-------|
| System ID | `aineff-ecs` |
| Package | `@aineff/ecs` |
| Cluster | Cluster 4 -- Audit & Death |
| Protocol Layer | Layer 3 (ORF) / Layer 4 (Enterprise Logic) |

## Purpose

ECS manages tamper-proof storage and chain-of-custody tracking for digital evidence
generated by AINEFF operations. It ensures that audit records, decision artifacts,
configuration snapshots, and communication logs are preserved in a forensically sound
manner. ECS supports legal holds, evidence production for litigation, and regulatory
examination with cryptographic proof of integrity.

## Standalone Revenue

**Tamper-proof evidence storage** -- managed service for enterprises needing legally
defensible digital evidence preservation for AI-driven operations.

## Interfaces

| Direction | Interface | Format |
|-----------|-----------|--------|
| IN | Evidence ingestion | Binary blobs + metadata |
| IN | Legal hold requests | REST API |
| OUT | Evidence packages | Signed, integrity-verified bundles |
| OUT | Chain-of-custody reports | Timestamped access logs |

## Events Emitted

- `ecs.evidence.sealed` -- new evidence committed to custody
- `ecs.hold.applied` -- legal hold placed on evidence set
- `ecs.evidence.produced` -- evidence exported for legal proceedings
- `ecs.evidence.released` -- evidence custody transferred or released
- `ecs.initialized` -- system startup complete
- `ecs.heartbeat` -- periodic health signal

## Events Consumed

- `acts.entry.recorded` -- audit entries become evidence candidates
- `ssdt.snapshot.captured` -- decision-time snapshots as evidence
- `npos.optout.requested` -- triggers evidence release procedures

## Closed-Loop Connections

| Connected System | Direction | Signal Type |
|-----------------|-----------|-------------|
| ACTS | <-- | Audit entries ingested as evidence |
| SSDT | <-- | Decision snapshots preserved |
| NPOS | <-- | Exit triggers evidence release |
| Audit Chain | --> | Anchors evidence hashes to blockchain |
| HOES | --> | Evidence requests routed to custodians |
| ACTS | --> | Logs all custody chain events |

## K8s Deployment

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: aineff-ecs
  namespace: aineff-system
  labels:
    app.kubernetes.io/name: ecs
    app.kubernetes.io/part-of: aineff
    aineff.io/cluster: audit-death
spec:
  replicas: 3
  selector:
    matchLabels:
      app: aineff-ecs
  template:
    spec:
      containers:
        - name: ecs
          image: ghcr.io/frankmax-com/aineff-ecs:latest
          ports:
            - containerPort: 3000
          resources:
            requests:
              cpu: 500m
              memory: 1Gi
            limits:
              cpu: "2"
              memory: 4Gi
```
